## 탐색 - 무정보 탐색 (Uninformed Search)

### Q1. 탐색 알고리즘을 평가하는 4가지 기준과 b, d, m의 의미를 설명하시오.
① 완전성(completeness): 해가 존재할 때 항상 찾는가 ② 최적성(optimality): 항상 최소비용 해를 찾는가 ③ 시간 복잡도(time complexity): 생성되는 노드 수 ④ 공간 복잡도(space complexity): 메모리에 유지되는 최대 노드 수. `b`=최대 분기계수(branching factor), `d`=최적해의 깊이(depth), `m`=상태공간의 최대 깊이(무한 가능).

### Q2. BFS와 DFS를 4가지 기준으로 비교하고, 각각 어떤 상황에서 유리한지 서술하시오.
BFS: FIFO 큐, 완전·최적(동일 단계비용), 시간 `O(b^d)`, 공간 `O(b^d)`. DFS: LIFO 스택, 불완전(무한 깊이)·비최적, 시간 `O(b^m)`, 공간 `O(bm)`. BFS는 해가 얕거나 무한 경로·루프가 많고 탐색공간이 작을 때 유리. DFS는 해가 많고 루프·무한경로가 없으며 메모리가 제약일 때 유리 — 공간이 선형이라는 점이 결정적 장점.

### Q3. IDS를 설명하고, "반복 탐색이 낭비"라는 비판이 왜 타당하지 않은지 수치 예로 설명하시오. IDS의 의의는?
깊이 제한 `L`=1부터 시작해 해를 찾을 때까지 `L`을 증가시키며 DFS를 반복한다. 노드 대부분이 마지막 깊이에서 생성되므로 중복 비용이 작다. `b`=10, `d`=5에서 N(IDS) ≈ 123,450, N(BFS) ≈ 111,110으로 약 11% 차이에 불과. 의의: DFS의 선형 메모리 `O(bd)`와 BFS의 완전성·최적성을 결합 → 탐색공간이 크고 해 깊이를 모를 때 선호되는 무정보 탐색.

### Q4. UCS의 동작 원리와, 완전성·최적성이 성립하는 조건과 이유를 서술하시오.
fringe에서 경로비용 `g(n)`이 최소인 노드를 항상 확장한다. 모든 단계비용이 `ε > 0` 이상이고 분기계수가 유한하면, 목표 비용에 도달하기까지 유한 번의 확장만 필요하므로 완전. 최적성(귀류법 서술): 비최적이라 가정하면 찾은 해보다 싼 목표가 존재해야 하는데, UCS는 정의상 더 싼 노드를 먼저 확장하므로 모순.

## 탐색 - 정보 이용 및 지역 탐색 (Informed & Local Search)

### Q5. Greedy best-first와 A*의 평가함수 차이를 쓰고 완전성·최적성을 비교하시오.
Greedy: `f(n) = h(n)`만 사용 → 목표에 가까워 보이는 노드만 추구, 방문기록이 없으면 루프에 빠져 불완전, 비최적(반례 존재), `O(b^m)`. A*: `f(n) = g(n) + h(n)`으로 실제 지불 비용과 남은 추정을 합산 → `h`가 admissible이면 완전·최적이며 optimally efficient(같은 휴리스틱으로 더 적게 확장하는 최적 알고리즘은 없음). 단, 최악의 시간·공간은 둘 다 지수적.

### Q6. Admissible heuristic을 정의하고 예를 들어 설명하시오. A*의 최적성과의 관계는?
모든 노드 `n`에 대해 `h(n) ≤ h*(n)`(실제 최적 비용)인 휴리스틱. 목표까지의 비용을 절대 과대평가하지 않는 낙관적 추정이다. 예: 도로 탐색의 직선거리는 실제 도로거리보다 클 수 없으므로 admissible. 정리: `h`가 admissible이면 tree-search A*는 최적해를 보장한다.

### Q7. 8-puzzle의 두 휴리스틱 h1, h2를 설명하고 값을 계산하시오. 어느 것이 더 좋은가, 근거는?
`h1`=제자리에 없는 타일 수(강의 예시 시작상태에서 8), `h2`=각 타일의 목표 위치까지 맨해튼 거리(Manhattan distance) 합(예시에서 3+1+2+2+2+3+3+2=18). 모든 `n`에서 `h2(n) ≥ h1(n)`이고 둘 다 admissible이므로 `h2`가 `h1`을 지배(dominate)하여 탐색에 더 좋다. 실증: `d`=12에서 IDS 3,644,035 / A*(h1) 227 / A*(h2) 73 노드.

### Q8. Relaxed problem이란 무엇이며 admissible 휴리스틱 생성과 어떤 관계인지 8-puzzle 예로 설명하시오.
행동에 대한 제약을 완화한 문제. 완화 문제(relaxed problem)의 최적해 비용은 원래 문제의 admissible 휴리스틱이 된다(제약이 적으니 비용이 같거나 작음). 8-puzzle에서 "타일이 아무 칸으로나 이동 가능"으로 완화하면 `h1`, "인접 칸으로 자유 이동"으로 완화하면 `h2`가 도출된다.

### Q9. Hill-climbing의 동작과 한계 3가지를 설명하고 해결책을 쓰시오. 8-queens에서의 성능 수치를 포함하라.
현재 상태의 이웃 중 값이 가장 좋은 쪽으로만 이동하고 정점에서 멈추는 greedy local search로, lookahead가 없다. 한계: local maxima, plateau(평지), ridge에서 정지. 해결: random restart(무작위 초기상태로 반복). 8-queens에서 14% 성공·86% 지역최소, 성공 시 평균 4스텝. `p`=0.14일 때 99% 성공을 위해 `(0.86)^n < 0.01` → 약 31회 재시작, 총 약 124 스텝.

## 적대적 탐색 (Adversarial Search)

### Q10. 게임(적대적 탐색)의 4가지 표준 가정과, 게임을 탐색 문제로 정의할 때의 4가지 구성요소를 쓰시오.
가정: 2인 교대 턴, 결정적(deterministic), 제로섬(zero-sum), 완전정보(perfect information). 구성요소: 초기 상태(보드 구성), Successor 함수(합법 수 목록), Terminal test(게임 종료 판정), Utility 함수(종료 상태의 수치적 가치 — 승 +1, 패 −1, 무 0).

### Q11. (트리 제시형) 주어진 2-ply 게임 트리에서 minimax 값을 구하는 과정을 서술하시오.
강의 예제: 리프 (3,12,8 / 2,4,6 / 14,5,2). 말단부터 위로 — MIN 노드들은 자식의 최소값을 취해 3, 2, 2가 되고, 루트(MAX)는 그 최대값 3을 취한다. 따라서 MAX의 최적 수는 왼쪽 자식(A1). Minimax는 MAX 입장에서 최악의 경우(상대 최적 응수)의 결과를 최대화한다. MIN이 비최적으로 두면 MAX의 결과는 minimax 값보다 나빠지지 않고 오히려 좋아질 수 있다.

### Q12. Alpha-beta pruning에서 α와 β의 의미를 정의하고, pruning 조건과 효과(복잡도)를 서술하시오.
`α`=DFS 경로상 MAX가 지금까지 확보한 최고값, `β`=경로상 MIN이 확보한 최저값. 탐색 중 어떤 노드의 값이 조상의 `α`·`β` 기준으로 이미 선택될 수 없음이 확정되면 남은 자식들을 잘라낸다. 결과는 minimax와 완전히 동일. 최악 `O(b^d)`, 최선(이상적 move ordering) `O(b^(d/2))` — 유효 분기계수가 √b로 감소(체스 35 → 약 6)하여 같은 시간에 약 2배 깊이 탐색 가능. 좋은 수 순서가 효율을 좌우한다.

### Q13. (트리 제시형) 리프가 3,4,1,2,7,8,5,6인 트리에서 어떤 노드가 prune되는가 — 풀이 절차를 쓰시오.
왼쪽부터 DFS로 진행하며 각 MIN 노드 완료 시 `α`를 갱신하고, 각 리프 평가 시 `β`를 갱신한다. 자식 평가 중 현재 MIN 노드의 `β ≤` 조상 MAX의 `α`가 되는 순간 그 MIN 노드의 남은 형제 리프들이 prune된다. 답안에는 각 노드 옆에 `α`·`β` 구간을 적어가며 "β ≤ α이면 중단" 규칙을 명시하는 것이 핵심 채점 포인트.

### Q14. 평가함수(evaluation function)란 무엇이며 체스의 예를 들어 설명하시오.
비종료 상태가 플레이어에게 얼마나 좋은지의 추정치. 보통 (자신 점수) − (상대 점수) 형태이며, 체스는 특징들의 선형 가중합 `Eval(s) = w1·f1(s) + … + wn·fn(s)`을 사용한다(예: `f1`=백 퀸 수 − 흑 퀸 수, `w1`=9). 제로섬이므로 한쪽에 `X`면 상대에겐 `−X`. 깊이 제한 탐색에서 terminal utility 대신 사용된다.

## 제약 충족 문제 (CSP)

### Q15. CSP를 정의하고, 호주 지도색칠 문제를 변수·도메인·제약으로 형식화하시오. Constraint graph란?
CSP는 상태가 변수 `Xi`와 도메인 `Di`의 값으로 정의되고, goal test가 변수 값 조합에 대한 제약 집합으로 주어지는 문제. 지도색칠: 변수={WA, NT, Q, NSW, V, SA, T}, 도메인={red, green, blue}, 제약=인접 지역 색 상이(WA ≠ NT 등). 해는 complete & consistent 할당. Constraint graph는 변수를 노드, (binary) 제약을 간선으로 표현한 그래프로 문제 구조 활용에 쓰인다.

### Q16. Backtracking search를 정의하고, 효율을 높이는 기법 4가지를 설명하시오.
백트래킹 = 한 노드에서 한 변수만 할당하는 DFS. 할당의 교환 가능성 덕분에 잎이 `d^n`으로 축소되며, CSP의 기본 무정보 알고리즘이다. 개선: ① MRV — 남은 합법 값이 최소인 변수 우선(실패를 빨리 발견) ② Degree heuristic — 동률 시 다른 변수에 제약을 가장 많이 거는 변수 선택 ③ LCV — 남은 변수의 선택지를 가장 덜 줄이는 값부터 시도 ④ Forward checking — 할당 시 미할당 변수의 합법 값을 추적, 도메인이 비는 즉시 백트랙(필연적 실패의 조기 감지). ①~③ 결합 시 1000-queens 가능.

### Q17. (추적형) 4-queens를 forward checking으로 풀 때 도메인 변화를 추적하시오.
X1=1 할당 → X2={3,4}, X3={2,4}, X4={2,3}. X2=3 → X3 도메인 공백 → 백트랙. X2=4 → 진행하다 모순 → X1=2로 백트랙. X1=2 → X2={4}, X3={1,3}, X4={1,3,4}. X2=4 → X3={1}, X4={3}. X3=1 → X4={3}. 해: X1=2, X2=4, X3=1, X4=3. 채점 포인트: 할당마다 줄어드는 도메인 기록 + 빈 도메인에서 즉시 백트랙.

### Q18. CSP에 local search를 적용하는 방법과 min-conflicts 휴리스틱을 설명하시오. 성능은?
모든 변수가 할당된 완전 상태에서 시작하되 제약 위반을 허용하고, 연산자는 변수 값 재할당. 변수 선택은 충돌 중인 변수에서 무작위, 값 선택은 min-conflicts(위반 제약 수를 최소화하는 값) = `h(n)`=위반 제약 총수에 대한 hill-climbing. 무작위 초기 상태에서 n-queens를 `n`=10,000,000 규모에서도 거의 상수 시간에 높은 확률로 해결한다.

## 의사결정트리 (Decision Tree)

### Q19. 의사결정트리를 정의하고 그 표현(내부 노드·가지·잎)을 PlayTennis 예로 설명하시오.
결정의 트리를 사용하는 분류·예측 지원 도구. 내부 노드=속성 테스트, 가지=속성 값, 잎=분류 값. PlayTennis 규칙 `if (Outlook=Sunny ∧ Humidity=Normal) ∨ (Outlook=Overcast) ∨ (Outlook=Rain ∧ Wind=Weak) then Yes else No`는 Outlook을 루트로, Sunny 가지 밑에 Humidity, Rain 가지 밑에 Wind를 테스트하는 트리와 동일하다. 본질은 큰 이질적 집단을 단순 분할 규칙의 연속으로 타깃 변수에 대해 균질한 소그룹으로 나누는 split rule의 집합.

### Q20. ★ 의사결정트리에서 가장 중요한 것은 무엇이며 그 이유는? (best split / purity)
가장 중요한 것은 각 노드에서의 최적 분할(best split) 선택이며, 그 평가 척도가 순도(purity)다. 최적 분할 = 데이터를 가장 균질한(homogeneous: 그룹 내 모든 데이터가 같은 타깃 값) 그룹으로 가장 잘 분리하는 분할, 즉 부분집합들의 purity를 가장 크게 증가시키는 분할이다. 좋은 분할은 또한 비슷한 크기의 노드를 만들고 아주 작은 노드를 만들지 않는다. 이유: 트리는 "어떤 속성으로 나눌까"의 반복으로 만들어지므로 분할 기준이 트리의 구조·크기·분류 성능을 모두 결정하고, Information Gain·Gini 등 모든 학습 알고리즘이 결국 purity 측정의 구현이기 때문.

### Q21. Entropy와 Information Gain을 정의(식 포함)하고, Transportation 데이터의 Gender 속성으로 Gain을 계산하시오.
`Entropy = Σ −pⱼ·log₂pⱼ` (불순도 측정, `pⱼ`=클래스 `j` 비율). `Information Gain(S,A) = Entropy(S) − Σᵥ(|Sᵥ|/|S|)·Entropy(Sᵥ)` — 원집합과 분할 부분집합들의 불순도 차이로, 매 노드에서 Gain 최대 속성을 선택한다. 계산: 전체(4B, 3C, 3T) `Entropy`=1.571(≈1.574). Male(3B, 1C, 1T) `Entropy`=1.371, Female(1B, 2C, 2T) `Entropy`=1.522 → `Gain(Gender)` = 1.574 − (5/10·1.371 + 5/10·1.522) = 0.125.

### Q22. (계산형) Transportation 예제에서 루트 노드 속성이 Travel Cost로 선택되는 과정과 트리 완성 과정을 서술하시오.
1차 반복에서 각 속성의 Information Gain을 비교: Gender 0.125, Car Ownership 0.534, Travel Cost 1.210, Income Level 0.695 → 최대인 Travel Cost가 루트. 분할 결과 Expensive → Car(3건), Standard → Train(2건)은 pure class라 잎이 되고, Cheap(4B, 1T, `Entropy`=0.722)만 추가 분할. 2차 반복(사용한 Travel Cost 제거)에서 Gender 0.322 > Car Ownership 0.171 = Income 0.171 → Gender 선택, Male → Bus(pure). 3차 반복에서 남은 2건은 Car Ownership(0 → Bus, 1 → Train)으로(또는 Income으로) pure하게 갈려 트리 완성.

### Q23. Gini 척도를 정의하고 예를 계산하시오. Information Gain과의 관계는?
`Gini = Σⱼ pⱼ²` — 클래스 비율 제곱의 합(population diversity). 균등 분포 루트 노드는 0.5² + 0.5² = 0.5(impure), 90:10 잎 노드는 0.1² + 0.9² = 0.82(거의 pure)로, 값이 클수록 순수하다. 분할 속성 결정에는 Information Gain 식의 `Entropy()`를 `Gini()`로 대체해 끼워 넣는 것만으로 충분하다(`Gain(S,A) = Σ(|Sᵥ|/|S|)·Gini(Sᵥ) − Gini(S)`).

### Q24. 의사결정트리의 장점 5가지와 단점 4가지를 서술하시오.
장점: ① 이해하기 쉬움 ② 비즈니스 규칙 집합으로 자연스럽게 매핑 ③ 다양한 실제 분류·예측 문제에 적용 가능 ④ 데이터에 대한 사전 가정 불필요 ⑤ 수치형·범주형 속성 모두 처리. 단점: ① 타깃(분류) 속성이 범주형이어야 함 ② 타깃 속성 1개(one class)로 제한 ③ 알고리즘이 때때로 불안정(local search와 유사) ④ 수치 속성 데이터셋의 트리는 매우 복잡해질 수 있음.

## k-최근접 이웃 (kNN)

### Q25. kNN 분류의 원리를 설명하고, 왜 "lazy learning"이라 부르는지 서술하시오.
객체를 속성 벡터 공간에서 가장 가까운 `k`개 사례의 다수결로 분류한다(`k`=1이면 최근접 이웃의 클래스). 모든 인스턴스를 `n`차원 유클리드 공간의 점으로 보고, 거리는 절대거리 `Σ|xᵢ−yᵢ|` 또는 유클리드 거리 `√Σ(xᵢ−yᵢ)²`를 사용한다. 분류 함수를 미리 전역적으로 학습하지 않고 국소적으로만 근사하며 모든 계산을 새 인스턴스가 도착하는 분류 시점까지 지연하므로 instance-based/lazy learning이다. 학습은 거의 비용이 없지만(숫자 변환·스케일링 정도) 분류는 `k`가 클수록 시간이 많이 든다.

### Q26. kNN에서 속성 스케일링(정규화)이 왜 필요한가? 식과 예를 들어 설명하시오.
각 차원의 크기(범위) 차이가 거리 계산을 왜곡해 큰 축이 이웃 판정을 지배하기 때문. `x' = (x − μ) / σ(x)`로 정규화하면 이 효과가 완화되고 분류 정확도가 자주 향상된다. 예(1-NN): X축 크기 6, Y축 크기 1인 공간에서 질의점은 '아마 white'로 분류되지만, 두 축을 정규화하면 '확실히 black'으로 분류가 바뀐다. 범주형 속성은 먼저 숫자로 변환한다(Outlook: Rain=0, Overcast=1, Sunny=2 등 → `<Sunny, Hot, High, Weak>` = `<2, 2, 1, 0>`).

### Q27. (추적형) PlayTennis 데이터에서 질의 {Sunny, Cool, High, Strong}을 K=3 kNN으로 분류하는 과정을 서술하시오.
① 범주형 → 숫자 변환 후 스케일링: 질의 = {1.18, −1.18, 1, 1}. ② 각 Day와의 유클리드 거리 계산 — 예: Day1과는 `√(0² + 2.36² + 0² + 2²)` = 3.09. ③ 최근접 3개 = Day2(No), Day7(Yes), Day11(Yes). ④ 다수결: `t_q = (0+1+1)/3 = 2/3 ≥ 1/2` → PlayTennis = YES. 채점 포인트: 변환·스케일링 → 거리 계산 → top-k 선정 → majority voting의 4단계 명시.

### Q28. Distance-Weighted Nearest Neighbor를 설명하고 Shepard's method란 무엇인지 쓰시오.
이웃들에게 질의로부터의 거리에 반비례하는 가중치(보통 거리의 역수 또는 역제곱)를 부여하는 방법. 질의에 가까운 이웃일수록 결과에 더 큰 영향을 주므로 직관적으로 타당하다. `K`를 전체 인스턴스 수로 두면 모든 인스턴스가 이웃이 되어 가중치를 갖고 결과에 기여하는데 이를 Shepard's method라 한다 — 단, 분류 시간 오버헤드가 크다. 이런 분류가 만드는 결정 표면의 예가 Voronoi diagram이다.

## 클러스터링 (Clustering)

### Q29. 클러스터링을 정의하고 지도학습과 비지도학습의 차이를 설명하시오. 클러스터링의 IR 응용 2가지는?
클러스터링 = 객체 집합을 유사한 객체들의 클래스로 그룹화하는 과정(같은 클러스터 내 유사, 클러스터 간 비유사). 가장 흔한 비지도 학습이다. 지도학습은 분류(label)가 주어진 데이터로, 비지도학습은 label 없는 raw data로 학습한다. 응용: ① recall 개선 — cluster hypothesis(같은 클러스터의 문서는 정보요구에 유사하게 반응)에 따라 질의에 맞은 문서 `D`와 같은 클러스터의 문서들도 반환("car" 질의가 "automobile" 문서 반환) ② 검색결과의 주제별 군집 내비게이션(그 외 Google News, 토픽 계층, 시각화).

### Q30. K-means 알고리즘의 절차, 종료 조건 3가지, 시간 복잡도를 서술하시오. "클러스터 불변"과 "centroid 불변" 조건은 동치인가?
절차: ① `K`개 무작위 시드 선택 ② 각 문서를 dist가 최소인 시드의 클러스터에 할당 ③ 각 클러스터의 centroid `μ(c) = (1/|c|)·Σx`로 시드 갱신 ④ 수렴까지 반복. 종료 조건: 고정 반복 횟수 / 모든 클러스터 불변 / 모든 centroid 불변 — 둘째와 셋째는 동치가 아니다(No). 수렴 근거는 K-means가 수렴이 증명된 EM의 특수 사례라는 점. 복잡도: 거리 `O(m)`, 재할당 `O(Knm)`, centroid `O(nm)`, `I`회 반복 → `O(IKnm)`로 완전 선형.

### Q31. K-means의 초기 시드 민감성 문제와 해결책 2가지를 예와 함께 서술하시오.
결과가 초기 무작위 시드에 따라 달라져, 나쁜 시드는 느린 수렴 또는 sub-optimal 클러스터링을 낳는다. 예: 같은 6점에서 B, E를 시드로 시작하면 {A,B,C}{D,E,F}로, D, F로 시작하면 {A,B,D,E}{C,F}로 수렴. 해결: ① 휴리스틱으로 좋은 시드 선택 — 이미 선택된 시드들로부터 가장 먼 문서를 다음 시드로 ② 서로 다른 시드로 여러 번 시도 — 이 알고리즘이 본질적으로 hill-climbing이기 때문.

### Q32. (비교분석) Single-link, Complete-link, Group average-link 클러스터링을 정의·갱신식·결과 성향 측면에서 비교하시오. IR에는 무엇이 더 적합한가?
Single-link: 두 클러스터의 유사도 = 가장 유사한 쌍의 유사도(갱신 `sim(AB,X) = max`). 각 멤버는 같은 클러스터의 최소 한 멤버와 임계 유사도를 넘으면 됨 → chaining effect로 소수의 크고 느슨한 클러스터, 불공평한 성장. Complete-link: 유사도 = 가장 덜 유사한 쌍(갱신 `min`). 각 멤버가 클러스터 내 모든 멤버와 임계 유사도를 넘어야 함 → 다수의 작고 단단한 클러스터. Group average-link: 쌍별 평균 사용, 모든 멤버가 유사도에 기여 — 둘의 절충으로 중간 정도 결합 강도. 일반적으로 complete-link이 IR에 더 적합하나 생성 비용은 single-link보다 크다.

### Q33. (추적형) A~F의 쌍별 유사도(AF 0.9, AE 0.8, BF 0.8, BE 0.7, AD 0.6, …)에서 single-link HAC의 병합 순서를 추적하시오.
항상 행렬에서 가장 큰 유사도 쌍을 병합하고 `sim(클러스터, X) = max` 규칙으로 행렬을 갱신한다. ① AF:0.9 병합 ② `sim(AF,E) = max(0.8, 0.3) = 0.8` → AEF ③ `sim(AEF,B) = 0.8` → ABEF(E와 B는 같은 레벨) ④ `sim(ABEF,D) = 0.6` → ABDEF ⑤ `sim(ABDEF,C) = 0.5` → 전체 병합. 임계값별 절단: 0.85에서 {AF}, {E}, {B}, {D}, {C} / 0.65에서 {AFEB}, {D}, {C} / 0.55에서 {AFEBD}, {C}. 채점 포인트: max 갱신식 명시와 단계별 행렬 축소.

## 강화학습 (Reinforcement Learning)

### Q34. 강화학습을 정의하고 특징 3가지를 쓰시오. 지도·비지도 학습과의 차이는?
반복적 시도의 시행착오 속에서 외부 환경으로부터 주어지는 Reward를 통해 Goal에 도달하는 방법을 배우는 기계학습(learn via experiences). 에이전트가 환경을 지각해 state·reward를 받고 action을 가하는 상호작용 구조. 특징: ① delayed reward — 선택한 action이 적합한지 당장 알 수 없음 ② 현재 action의 결과를 정확히 몰라도 무관 ③ life-long learning에 활용 가능. 차이: 지도학습은 목표값(label)이 제시된 데이터, 비지도는 label 없는 데이터로 학습하지만, 강화학습은 명시적 정답 대신 행동에 대한 보상 신호로 행동을 점차 강화한다.

### Q35. Q-Learning에서 Q(s,a)의 정의·갱신식·학습 후 행동 선택 방법, 그리고 전체 알고리즘을 서술하시오.
`Q(s,a)`는 상태 `s`에서 행동 `a`를 선택하는 것이 유리한 정도를 나타내는 추정 효용함수로, 즉각 보상과 다음 상태의 잠재 보상 최대값의 합으로 정의: `Q(s,a) = r(s,a) + γ·max_a′ Q(s′,a′)` (`γ ∈ [0,1]`=보상 전파 계수, `s′`=`a` 선택 후 새 상태). 학습 완료 후엔 `π(s) = argmax_a Q(s,a)`로 행동 선택. 알고리즘: 모든 `(s,a)`의 `Q`를 0으로 초기화하고 수렴까지 ① 현재 `s`에서 가능한 임의 `a` 선택·실행 ② immediate reward 수령 ③ 새 상태 `s′` 감지 ④ 갱신식 적용 ⑤ `s = s′` 반복. 학습 시점엔 평가가 미완이라 당시 최적으로 평가된 action이 실제 최적이 아닐 수 있고, reward를 역방향으로 전파하며 학습한다.

### Q36. (계산형) 6칸 격자(목표 S6, 도달 보상 100, γ=0.5)에서 Q값이 전파되는 과정을 3개 iteration으로 보이시오.
Iter1(S1 → S2 → S3 → S6): `Q(S1,a12) = 0 + 0.5·max(0,0,0) = 0`, `Q(S2,a23) = 0`, 최종 상태 도달 시 `Q(S3,a36) = r = 100`. Iter2(S2 → S3 → S6): `Q(S2,a23) = 0 + 0.5·max(Q(S3,a32), Q(S3,a36)) = 0.5·100 = 50`. Iter3(S4 → S1 → S2 → S3 → … → S5 → S6): `Q(S4,a41) = 0`, `Q(S1,a12) = 0.5·50 = 25`, `Q(S3,a32) = 0.5·50 = 25`, `Q(S5,a56) = 100`. 핵심: 보상이 목표 인접 상태부터 `γ`배씩 감쇠하며 시작 쪽으로 전파. 수렴 후: 목표 인접 100, 두 칸 거리 50, 세 칸 25, 네 칸 12.5.

## 인공신경망 (ANN)

### Q37. Perceptron의 구조와 Widrow-Hoff rule(delta rule)에 의한 학습 절차를 서술하시오.
생물 뉴런의 Dendrites(입력)·Nucleus(처리)·Axon terminals(출력)를 모방한 ANN의 기본 unit. 복수 입력 `pᵢ`와 가중치 `wᵢ`의 곱의 합이 임계치 `θ` 기준으로 활성화 함수(hard limiter/linear/sigmoid)를 거쳐 출력 `y`의 흥분 여부를 결정. 학습(지도): ① `W`·`θ`를 작은 임의값으로 초기화 ② 입력 패턴 `X`와 목표 출력 `d(t)` 제시 ③ `y(t) = fₙ(ΣWᵢXᵢ(t) − θ)` 계산 ④ `Wᵢ(t+1) = Wᵢ(t) + α·[d(t) − y(t)]·Xᵢ(t)`로 갱신 — 출력과 목표가 같으면 가중치 불변, 다르면 차이를 줄이는 방향으로 변경.

### Q38. 단층 Perceptron이 AND는 구현하지만 XOR은 구현하지 못하는 이유를 식으로 보이고, 해결책을 서술하시오.
AND(`θ`=0.5): `0·W0 + 0·W1 = 0 < 0.5`, `W1 < 0.5`, `W0 < 0.5`, `W0 + W1 > 0.5`는 `W0 = W1 = 0.3`(또는 0.4)으로 충족 가능. XOR: `0 < 0.5`, `W1 > 0.5`, `W0 > 0.5`, `W0 + W1 < 0.5`를 동시에 만족하는 `W0, W1`은 존재하지 않음 — XOR은 linearly non-separable이라 하나의 퍼셉트론(직선 하나)으로 분리 불가. 해결: 2~3개 층의 다층 퍼셉트론(MLP) 사용 — 3층 Perceptron으로 어떤 문제도 근사적으로 해결 가능하며, 이것이 Backpropagation NN의 기반이다.

### Q39. Backpropagation Neural Network의 구조와 학습 원리(Error backpropagation + Gradient Descent)를 설명하고 문제점을 쓰시오.
입력층과 출력층 사이에 하나 이상의 은닉층을 둔 단방향 신경망으로, 단층의 선형분리 한계를 해결하고 일반적 연속함수 근사에 널리 쓰인다. 학습: 목표값 `d`와 출력 `o`의 오차제곱합 `E = ½·Σ(d − o)²`를 Gradient Descent로 최소화 — 출력층 오류 `δ_pk = (d_pk − O_pk)·O_pk·(1 − O_pk)`로 은닉-출력 가중치를 `W_jk(t+1) = W_jk(t) + η·δ_pk·O_pj`로 갱신하고, 오류를 역전파해 은닉층 오류 `δ_pj = [Σδ_pk·W_jk]·O_pj·(1 − O_pj)`로 입력-은닉 가중치를 갱신한다(일반화된 델타 규칙, sigmoid의 미분 `y(1 − y)` 이용). 모든 학습쌍에 반복하며 `E`가 허용값 이하 또는 최대 횟수 도달 시 종료. 문제점: 상위층 오류를 하위층으로 역전파하는 것은 생물학적 현상과 불일치(하위 뉴런은 상위 목표값을 모름) — 그럼에도 가장 보편적인 ANN 학습법이다.
